### 프로세스 상태

프로세스란 현재 실행 중이거나 곧 실행이 가능한 PCB를 가진 프로그램이다.

하나의 작업이 접수되어 완료될 때까지는 다음과 같은 여러 상태의 변화를 거치게 된다.

```
1. 실행 상태(running) : 프로세스가 중앙처리장치를 차지하고 있는 상태

2. 준비완료 상태(ready) : 중앙처리장치가 사용 가능하게 될 때 그것을 할당받을 수 있는 상태

3. 대기 상태(block) : 프로세스가 중앙처리장치를 차지하고 처리하다가 입출력 처리 등을 하게 되면,
                      중앙처리장치를 양도하고 입출력 처리가 완료될 때까지 기다리는 상태
```

중앙처리장치는 단 하나만의 프로세스가 실행되고 있지만, 다른 프로세스들은 준비완료 상태나 대기 상태로 있을 수 있고,

따라서 준비완료 상태의 프로세스들과  대기 상태의 프로레스들을 위해 각각 준비완료 리스트(큐)와 대기 리스트(큐)를 만들고, 

우선순위에 의해 준비완료 리스트에 있는 첫 번째 프로세스에게중앙처리장치를 할당할 수 있게 한다.

프로세스 상태의 전이는 다음과 같이 나타내는데

```
제시, 보류, 준비완료, 실행, 대기, 완료 

1. 제시 -> 보류 : 작업이 시스템에 제시되면 스풀러에 의해 디스크에 수록되어 보류 상태가 된다.

2. 보류 -> 준비완료 : 시행될 프로레스로 만들어 준비완료 리스트의 가장 뒤에 삽입, 
준비완료 리스트에 보내는 역할은 작업 스케줄러가 담당.

3. 준비완료 -> 실행 : 준비완료 상테에 있는 여러 프로세스 중 실행될 프로세스를 선정하여
중앙처리장치에 할당하고, 프로세스는 준비완료 상태에서 실행 상태로 변환(dispatch)

4. 실행 -> 준비완료 : 프로세스가 할당시간이 지나도록 계속 중앙처리장치를 차지해야할 경우, 
그 프로세스 한으로 중앙처리장치를 일단 포기하여 준비완료 상태로 변환

5. 실행 -> 대기 : 실행 상태의 프로세스 할당시간을 초과하기 전에 입출력이 발생하면 중앙처리장치를 양도하고,
입출력이 완료될 때까지 대기 상태로 변환(block)

6. 대기 -> 준비완료 : 입출력이 완료되면 준비완료 상태로 전환사키는데 
이러한 여러 가지 프로세스의 상태에 대한 조사와 통보는 트래픽 제어기(traffic controller)가 한다.
```
