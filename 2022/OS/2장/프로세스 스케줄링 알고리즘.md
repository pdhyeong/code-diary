### FCFS(First Come First Served) 스케줄링

비선점 스케줄링 방법중 하나이고, 가장 간단한 스케줄링 방식이다. 프로세스들이 대기 큐에 도착한 순서대로 중앙처리장치를 할당받는다.

중앙처리장치를 차지하면 완료될 때까지 CPU를 점유하며 수행

요즘은 거의 사용하지 않고 가끔 사용된다. 대화식에서는 더욱 사용하기 힘든데 반응속도가 느리기 때문이다.

```
평균반환 시간 = 첫 프로세스 버스트 반환시간 {a + (a  + 두번째 프로세스 반환시간 b) + (b + 세번째 프로세스 반환시간 c)} / 프로세스 갯수 이다.

평균 대기시간 = (0 + a + b) / 프로세스 갯수 
```

### SJF(Shortest Job First) 스케줄링

비선점 스케줄링 방식으로 수행시간이 가장 짧은 것 먼저 수행한다. 

설명처럼 FCFS보다 시간이 크게 단축될 것처럼 예상된다. 하지만 평균 대기시간을 감소시키는 반면 대기시간의 분산이 예측하기가 어렵다.

그러므로 사용자에 의존하는 비중이 높아진다. 

시행시간에 따라 예측된 시간보다 더 오랫동안 수행되면 중단되고, 중단된 프로세스에 대해서도 수수료를 부과하는 것을 사용자가 미리 알도록 경고함으로써 문제를 해결하는 경우도 있다.

SJF도 FCFS처럼 비선점 방식을 이요하므로 응답시간이 보장되어야 하는 시분할 시스템에는 적당하지 않다.

```
SJF 평균 반환시간 = 버스트시간이 가장 짧은 프로세스의 버스트시간 a + (a + 버스트시간이 두번째로 짧은 프로세스의 버스트시간 b) + (b + 세번쨰...) / 프로세스의 갯수 
```

### 우선순위(Priority) 스케줄링

일반적인 우선순위 스케줄링 알고리즘의 특별한 경우이다.

우선순위가 프로세스마다 주어지고, 중앙처리장치는 가장 높은 우선순위를 가진 프로세스로 할당된다.

SJF 알고리즘은 단순한 우선순위 알고리즘으로 우선순위는 다음 중앙처리 장치 버스트의 역이고, 중앙처리장치 버스트가 클스록 우선순위가 낮으며, 반대도 성립한다.

|프로세스|버스트시간|우선순위|
|----|-----|-----|
|P1|8|2|
|P2|2|3|
|P3|1|1|
|P4|2|3|
|P5|6|4|


위의 표에서 평균반환시간과 평균 대기시간을 구해보면 다음과 같다.

```
P3  P1   P2    P4    P5

0  1    9   11   13   19
```

평균 반환시간: (9+11+1+13+19)/5 = 10.6

평균 대기시간: (1+9+0+11+13)/5 = 6.8

우선순위는 내부요인과 외부요인의 의해서 정의 할 수 있고,

우선순위 스케줄링은 선점이나 비선점 될 수 있는데,

선점 우선순위 스케줄링은 새로 도착한 프로세스의 우선순위가 현재 실행되는 프로세스의 우선순위보다 높으면 중앙처리장치를 중앙처리장치를 선점하고,

비선점 우선순위 스케줄링 알고리즘은 단순히 준비완료 큐의 선두부분에 새로운 프로세스를 넣은 후 중앙처리장치의 할당을 기다린다.

이러한 우선순위 스케줄링 알고리즘의 주요 문제는 무한대기(indefinte blocking) 또는 기아현상(starvation)이다.

실행준비는 되어 있는데 중앙처리장치를 사용하지 못하는 프로세스는 중앙처리장치를 대기한다. 

이 우선순위 스케줄링 알고리즘은 낮은 우선순위의 프로세스들이 중앙처리장치를 무한히 대기하게 하는 경우를 발생시킬 수 있다.

해결방법으로는 에이징(aging)이 있다.

대기중인 프로세스들의 우선순위의 대기시간정도를 정해 점진적으로 1씩 증가 시켜 우선순위를 높여 해결하는 방식이다.

### 라운드 로빈(Round-Robin) 스케줄링

라운드로빈 스케줄링은 특히 시분할 시스템을 위하여 고인된 선점 스케줄링 방식의 하나다.

FCFS의 의해서 내보내지며 각 프로세스는 같은 크기의 중앙처리장치 시간을 할당받는다.

보통 할당 시간은 10ms ~ 100ms 사이이다.

시간 내에 완료하지 못하면 다음 프로세스가 선점되고 실행 중이던 프로세스는 준비완료 큐의 가장 마지막으로 간다.

시분할 시스템에 효과적이고, 효율적인 context switching 기법을 쓰면서 주기억장치 내에 유지될 수 있도록 충분한 기억용량을 준비한다면 선점으로 인한 오버헤드를 최대한 줄일수있다.

할당시간의 크기는 시스템의 효율적인 동작에 절대적인 영향을 미친다.

예를 들어 시스템이 할당시간을 매우 큰 것과 적은 것을 취했을 경우 생각해보면

할당 시간이 너무 크면 각 프로세스는 그 프로세스를 처리 완료하는 데 필요한 시간보다 많은 시간이 할당되는 경우가 되므로 이때의 라운드 로빈 방법은 FCFS와 같아진다.

반면 시간이 너무 적으면 문맥 교환을 위한 분배하는 데 소모되므로 실제적으로 사용자의 연산을 실행하는 데 보내는 시간이 적어지게 되어 시스템의 성능이 떨어지게 된다.
